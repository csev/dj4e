Django Sessions: Cookies, Middleware, and Database Storage

Django, session middleware, session cookies, session identifier, database storage, session data

Django's session middleware uses a session cookie containing a unique session identifier (a large random number) to associate incoming HTTP requests with server-stored session data. This mechanism allows the server to maintain user-specific state across multiple requests without requiring the browser to send additional data for each request.

By default, Django stores sessions in a database table (e.g., `django_session`). Session data is a dictionary serialized to JSON and encoded in Base64 for safe storage. Developers access session data via `request.session` in view functions to store, retrieve, and delete key-value pairs.
