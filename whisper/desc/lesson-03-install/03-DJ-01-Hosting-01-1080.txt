Django Application Structure

django, project, application, folder_structure, settings, urls, wgsi, models, views, mvc

This lecture explains the folder structure and key files of a Django application, distinguishing between projects (which are websites) and applications (reusable components). It describes the typical project layout, including the project directory (e.g., dj4esamples/dj4esamples) and applications (e.g., hello), and highlights critical files such as settings.py, urls.py, wgsi.py, models.py, views.py, and admin.py.

The lecture details the request-response cycle in Django, where a browser request is routed by urls.py to a view in views.py. The view handles data through models and generates a response using templates, following the Model-View-Controller pattern.
