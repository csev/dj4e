Django Migrations: Portable Database Evolution Process

Django migrations, database evolution, models.py, migration files, database portability, SQLite

Django migrations are Python-based files generated by `make migrations` that describe incremental database schema changes. These files are portable across database types like SQLite and MySQL, and follow a versioned sequence starting from 001_initial.py. Each migration file represents the evolution from previous models.py definitions without wiping existing databases.

The `migrate` command applies these migration files in order to update the database schema, tracking applied migrations in a Django Migrations table. In SQLite environments, the database can be reset by deleting the SQLite file (e.g., `rm db.sqlite`), though this requires recreating admin accounts and data.
