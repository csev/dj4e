Django Migrations: Creating and Applying Database Schema Changes

Django, migrations, database_schema, portability, migrate

Django migrations are generated by reading models.py and settings.py to describe database schema changes. They are portable across database backends because they provide a logical description of the schema without specific SQL commands.

The migrate command applies migrations in order to update the database, and Django maintains a table to record which migrations have been applied. In development, resetting migrations can be achieved by deleting the migration files and re-running make migrations; for SQLite, the entire database can be reset by deleting the database file.
